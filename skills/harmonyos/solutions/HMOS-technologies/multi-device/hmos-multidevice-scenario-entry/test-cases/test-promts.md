# Scenario Entry Test Prompts

用于验证总场景入口的三层测试能力。

## 1. Smoke Prompts

### 1. 窗口尺寸变化

一个平板详情页在旋转和拖拽窗口后布局没有刷新，需要补窗口尺寸监听和同步机制。请给出开发阶段的处理建议。

### 2. 折叠态与折痕

折叠屏应用在悬停态下，上下两个区域的内容被折痕切开，需要重新设计折叠态布局和折痕避让方式。请先给出需求分析设计。

### 3. 软键盘与系统区域避让

线上反馈：评论输入框在弹出软键盘后被完全遮住，同时顶部挖孔区域也压住了标题。请给出问题修复思路。

### 4. 多输入与焦点交互

桌面端页面需要同时支持鼠标 hover、右键菜单和键盘焦点导航，并确保触摸和鼠标的交互一致。请给出开发阶段方案。

### 5. 自然方向与旋转语义

设备在横竖屏切换后方向判断不稳定，需要明确自然方向、rotation 值和 setPreferredOrientation 的语义边界。请先给出需求分析设计。

### 6. 硬件能力与外设接入

应用需要根据设备是否支持蓝牙外设和摄像头能力做差异化接入，并在能力不足时提供降级路径。请给出开发阶段建议。

### 7. 复合场景

给一个折叠屏视频详情页做多设备适配：展开态双栏，悬停态上下分屏，评论输入框不被键盘遮挡，旋转后布局同步更新。请给出开发阶段的方案。

## 2. Scene-Functional Prompt

给一个折叠屏视频详情页做多设备适配：展开态双栏，悬停态上下分屏，评论输入框不被键盘遮挡，旋转后布局同步更新。请不要直接写代码，先给出可交给下游多设备场景处理的场景拆分与交接结果。

## 3. Real-World Prompt

发版前回归发现：折叠屏视频详情页在展开态双栏正常，但切到悬停态后评论区被压到折痕附近，弹出键盘后输入框看不见，旋转设备后播放器和评论区比例还会错位。请先给出修复分流方案，说明应该由哪些多设备场景接手。

## 典型验证点

- `primary_phase` 是否正确
- `primary_scene` 是否正确
- `secondary_scenes` 是否覆盖复合问题
- `next_scene_refs` 是否指向正确的场景文件
- 输出字段是否完整，尤其是阶段输出契约字段
- 场景交接是否足够让下游 skill 继续执行
