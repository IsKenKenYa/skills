# 相机旋转角度术语基线

来源：[华为开发者文档 - 相机旋转角度术语](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/camera-rotation)

## 目标

方向类问题最容易先在角度概念层出错。本文档定义设备自然方向、屏幕旋转角度、镜头安装角度、预览旋转角度等核心术语，作为旋转适配的概念基线。

## 使用场景

| 场景 | 触发条件 | 推荐用法 | 为什么要判断 | 不判断的风险 |
| --- | --- | --- | --- | --- |
| 预览旋转概念理解 | 需要理解为什么预览方向不对 | 查阅"预览旋转角度"公式 | 角度概念混淆是旋转问题的常见根因。 | 盲目调参，越调越乱。 |
| 拍照方向异常排查 | 照片方向与预期不符 | 查阅"拍照旋转角度"公式 + 前后置区分 | 前置用减法，后置用加法，混淆即出错。 | 修复后仍然方向错误。 |
| Surface 比例异常 | 画面拉伸/压缩 | 查阅屏幕旋转角度与 Surface 比例关系 | 0°/180° 和 90°/270° 规则不同。 | 比例始终对不上。 |

## 术语速查表

| 术语 | 定义 | 获取方式 |
|------|------|---------|
| 设备自然方向 | 设备默认使用方向 | 设备固有属性 |
| 屏幕旋转角度 | 从自然方向到当前方向的顺时针夹角 | `Display.rotation`（枚举值 ×90°） |
| 相机镜头安装角度 | 采集图像方向到自然方向的顺时针夹角 | 后置 90°，前置 270° |
| 预览旋转角度 | 镜头安装角度 + 屏幕显示补偿角度 | `previewOutput.getPreviewRotation()` |
| 拍照旋转角度 | 后置：镜头安装角度 + 重力方向；前置：镜头安装角度 - 重力方向 | `photoOutput.getPhotoRotation(deviceDegree)` |
| 录像旋转角度 | 后置：镜头安装角度 + 重力方向；前置：镜头安装角度 - 重力方向 | `videoOutput.getVideoRotation(deviceDegree)` |

## 核心术语

### 设备自然方向

设备默认的使用方向。

- **手机**：竖屏且充电口向下
- **平板**：横屏

### 屏幕显示方向

屏幕显示方向指当前用户视角下，设备正确的显示方向。

### 屏幕旋转角度

显示设备的屏幕顺时针旋转角度，即设备从自然方向到当前方向的顺时针夹角。

通过 `Display.rotation` 获取，枚举值与角度的对应关系：
- `Display.rotation = 0` → 0°
- `Display.rotation = 1` → 90°
- `Display.rotation = 2` → 180°
- `Display.rotation = 3` → 270°

### 相机镜头安装角度

相机镜头安装角度指相机采集图像方向到设备自然方向在顺时针方向的夹角。

以手机为例，手机后置相机传感器是横屏安装的，当手机在竖屏方向使用后置相机镜头拍摄时，相机采集到的原始图像需要顺时针旋转90度才能与设备自然方向保持一致，所以后置相机的镜头角度为90度。

而手机前置镜头是朝向使用者的，当手机在竖屏方向使用前置相机镜头拍摄时，出图方向与后置出图方向互为镜像，前置相机的镜头角度为270度。

- **后置相机镜头角度 = 90°**
- **前置相机镜头角度 = 270°**

### 预览旋转角度

> 在实际开发过程中，推荐通过接口获取预览旋转角度。

在预览时，图像旋转角度与屏幕显示旋转角度相关。系统将以原始图像方向为基线，根据相机镜头角度和屏幕显示补偿角度，旋转图像。

**计算公式**：`图像旋转角度 = 镜头安装角度 + 屏幕显示补偿角度`，屏幕显示补偿角度的值与屏幕旋转角度相等。

计算的角度设置给系统侧，作用于直接送显场景，应用自绘制请参考应用自绘制预览角度处理。

#### 后置相机预览角度示例

| 设备条件 | 镜头角度 | 屏幕旋转角度 | 预览旋转角度 |
|---------|---------|------------|------------|
| 竖屏充电口向下 | 90° | 0°（`Display.rotation = 0`） | 90° + 0° = 90° |
| 横屏充电口向左 | 90° | 90°（`Display.rotation = 1`） | 90° + 90° = 180° |
| 竖屏充电口向上 | 90° | 180°（`Display.rotation = 2`） | 90° + 180° = 270° |
| 横屏充电口向右 | 90° | 270°（`Display.rotation = 3`） | 90° + 270° = 0° |

#### 前置相机预览角度示例

前置相机有镜像出图特性。

| 设备条件 | 镜头角度 | 屏幕旋转角度 | 预览旋转角度 |
|---------|---------|------------|------------|
| 竖屏充电口向下 | 270° | 0°（`Display.rotation = 0`） | 270° + 0° = 270° |
| 横屏充电口向左 | 270° | 90°（`Display.rotation = 1`） | 270° + 90° = 0° |
| 竖屏充电口向上 | 270° | 180°（`Display.rotation = 2`） | 270° + 180° = 90° |
| 横屏充电口向右 | 270° | 270°（`Display.rotation = 3`） | 270° + 270° = 180° |

### 应用自绘制预览角度处理

应用自绘制场景是指应用获取图片后，通过libyuv、GL等图形处理库进行二次处理，生成新的图像数据并送到显示设备进行渲染绘制。

常见的实现方式是通过使用ImageReceiver创建的回调流，应用层作为消费端，自行处理图片旋转等操作，以适应自绘制场景的预览角度需求。自绘制场景预览角度与预览旋转角度中描述的场景存在细微差异。

主要差异体现在使用前置镜头拍摄预览的场景：

自绘制场景可以按照预览旋转角度中的图示方式，先根据镜头的安装角度进行旋转，随后进行镜像处理，最后再次旋转以适应屏幕角度。然而，这种方式包含多个步骤，较为繁琐，不被推荐。

自绘制场景也可以采取先旋转再镜像的方式，这种方式需要考虑水平镜像和垂直镜像。

#### 前置相机自绘制示例

| 设备条件 | 镜头角度 | 屏幕旋转角度 | 预览旋转角度 |
|---------|---------|------------|------------|
| 竖屏充电口向下 | 270° | 0°（`Display.rotation = 0`） | 270° + 0° = 270° |
| 横屏充电口向左 | 270° | 90°（`Display.rotation = 1`） | 270° + 90° = 0° |

### 拍照/录像角度

在拍照、录像时，图像旋转角度与设备重力方向（即设备旋转角度）相关。

- **使用后置相机拍摄**：`图像旋转角度 = 镜头安装角度 + 重力方向`
- **使用前置相机拍摄**：`图像旋转角度 = 镜头安装角度 - 重力方向`

#### 后置相机拍照/录像角度示例

| 设备条件 | 镜头角度 | 设备旋转角度 | 图像旋转角度 |
|---------|---------|------------|------------|
| 横屏充电口向左 | 90° | 90° | 90° + 90° = 180° |

应用需要监听 `SensorId.GRAVITY` 事件，获取重力传感器在x、y、z三个方向上的数据，计算得出设备旋转角度，参考计算设备旋转角度。

## 决策规则

- **预览角度公式**：`图像旋转角度 = 镜头安装角度 + 屏幕显示补偿角度`。屏幕显示补偿角度的值与屏幕旋转角度相等。
- **后置相机拍照/录像角度公式**：`图像旋转角度 = 镜头安装角度 + 重力方向`。
- **前置相机拍照/录像角度公式**：`图像旋转角度 = 镜头安装角度 - 重力方向`（前置用减法而非加法，这是最常见的出错点）。
- **屏幕旋转角度获取**：`Display.rotation` 枚举值（0/1/2/3）分别对应 0°/90°/180°/270°。
- **镜头安装角度默认值**：后置 90°，前置 270°。
- **自绘制前置场景**：推荐"先旋转再镜像"方式。需考虑水平镜像和垂直镜像的差异。

## 约束

- 预览旋转角度推荐通过 `getPreviewRotation()` 接口获取，而非手动计算。
- 拍照/录像角度推荐通过 `getPhotoRotation()`/`getVideoRotation()` 接口获取。
- 后置与前置的角度计算公式不同：拍照/录像时后置用加法（+重力方向），前置用减法（-重力方向）。
- 自绘制场景的前置处理与直接送显场景存在细微差异，不可混淆。
- 本文档定义的公式为概念基线，实际开发应结合 [相机旋转角度适配指南](camera-rotation-adaptation.md) 中的 API 进行实现。
