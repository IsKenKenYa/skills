# hmos-atomicservice-assistant Skill 测试提示词

---

## 基础功能测试

### 测试场景 1：新建元服务项目引导

**提示词**：
```
我想从零开始创建一个鸿蒙元服务，请问需要哪些步骤？有什么特别需要注意的地方？
```

**预期输出**：
- AI 代理识别"新建元服务"意图，走新建分支而非改造分支
- 输出包含 `bundleType: "atomicService"` 配置说明
- 输出包含包大小限制（单分包 ≤ 2MB，总包 ≤ 10MB）
- 输出提及只能使用带 `@atomicservice` 标注的 API
- 输出包含隐私托管和可信域名配置提示
- 流程步骤完整，逻辑有序

---

### 测试场景 2：识别元服务触发关键词

**提示词**：
```
我想做一个免安装的小程序，用户不需要下载就能直接使用，鸿蒙有没有类似的能力？
```

**预期输出**：
- AI 代理识别"免安装"关键词，正确关联到元服务（Atomic Service）
- 输出向用户介绍元服务的概念和核心特性
- 输出包含元服务与普通应用的主要区别
- AI 代理引导用户进入新建或改造的决策流程

---

## 合规检查测试

### 测试场景 3：上架前完整合规检查

**提示词**：
```
我的元服务开发完了，准备提交审核，能帮我做一个上架前的合规检查吗？
```

**预期输出**：
- AI 代理输出分级合规检查清单（Critical 和 Warning 分开列出）
- Critical 项包含（共 9 项）：
  - `bundleType` 检查
  - `@atomicservice` API 限制检查
  - 可信域名配置检查
  - 隐私托管检查
  - `getQuickLoginAuthCode` 静默登录检查
  - 包大小限制检查（2MB / 10MB）
  - `AtomicServiceEnhancedWeb` 组件检查
  - 跳转规范检查
  - Navigation 路由检查
- Warning 项包含：睫毛图标、Menubar 避让、分享功能、ICP 备案

---

### 测试场景 4：针对特定配置的合规问题定位

**提示词**：
```
我的 app.json5 配置如下：
{
  "app": {
    "bundleName": "com.example.myapp",
    "bundleType": "app",
    "vendor": "example"
  }
}
这个配置有什么问题吗？
```

**预期输出**：
- AI 代理识别 `bundleType: "app"` 与元服务要求不符
- 明确指出需要将 `bundleType` 修改为 `"atomicService"`
- 提示修改后需同步检查其他合规项
- 不仅仅报错，同时给出正确的配置示例

---

## 技术问题测试

### 测试场景 5：H5 组件使用问题

**提示词**：
```
我在元服务里用 ArkWeb 加载网页，运行时报错，提示组件不支持，这是什么原因？怎么解决？
```

**预期输出**：
- AI 代理明确指出元服务不支持 `ArkWeb` 组件
- 给出替换方案：使用 `AtomicServiceEnhancedWeb` 组件
- 提供组件替换的基本用法说明或代码示例
- 不应模糊回答或让用户自行查找文档

---

### 测试场景 6：静默登录接口问题

**提示词**：
```
我在元服务里调用华为账号的登录接口，但是提示"该 API 在元服务中不可用"，请问元服务里怎么实现账号登录？
```

**预期输出**：
- AI 代理识别"静默登录"问题
- 明确指出应使用 `getQuickLoginAuthCode` 接口
- 说明静默登录的适用场景（用户已登录华为账号的情况下）
- 提供调用示例或关键参数说明

---

### 测试场景 7：网络请求被拦截问题

**提示词**：
```
我的元服务向后端发送 HTTP 请求，域名是 api.mycompany.com，但请求一直被拦截，怎么处理？
```

**预期输出**：
- AI 代理识别"可信域名"问题
- 说明元服务网络请求需要在华为开发者联盟配置可信域名白名单
- 给出配置可信域名的操作路径（华为开发者联盟 → 元服务 → 可信域名）
- 提示配置生效可能有延迟

---

### 测试场景 8：包大小超限解决方案

**提示词**：
```
我的元服务编译后，主分包大小是 3.5MB，超出了限制，有什么办法可以把包压下来？
```

**预期输出**：
- AI 代理确认单分包限制为 2MB
- 给出至少两种减包方案，例如：
  - 将资源文件（图片/字体）托管到网络，按需加载
  - 将功能拆分为多个分包（每个分包 ≤ 2MB，总包 ≤ 10MB）
  - 压缩图片资源
  - 移除未使用的依赖库
- 输出应有具体可操作的建议，而非泛泛而谈

---

## 上架审核测试

### 测试场景 9：审核驳回原因分析

**提示词**：
```
我的元服务提交审核被拒了，驳回原因是："1. 应用图标不符合元服务规范；2. 存在不受支持的 Web 组件；3. 隐私政策未托管至华为平台。"请帮我逐条分析如何修复。
```

**预期输出**：
- AI 代理逐条响应三个驳回原因，不遗漏
- 问题 1（图标）：说明睫毛图标规范，左上角白色弧线标识的设计要求
- 问题 2（Web 组件）：明确指出 `ArkWeb` → `AtomicServiceEnhancedWeb` 替换
- 问题 3（隐私托管）：给出华为开发者联盟隐私托管配置步骤
- 建议修复后重新执行合规检查再提交

---

### 测试场景 10：上架常见被拒原因预防

**提示词**：
```
元服务审核经常因为哪些原因被拒？我想提前避免这些问题。
```

**预期输出**：
- AI 代理列出常见审核被拒原因，至少包含：
  - 图标不符合睫毛规范
  - `ArkWeb` 未替换为 `AtomicServiceEnhancedWeb`
  - 隐私政策未托管
  - 可信域名未配置
  - `bundleType` 配置错误
  - 包大小超限
  - 跳转了不允许跳转的目标（普通应用）
- 输出具有预防性建议，而非仅列举问题

---

## 边界条件测试

### 测试场景 11：意图模糊时的追问行为

**提示词**：
```
帮我搞个元服务
```

**预期输出**：
- AI 代理不直接给出全量输出，而是先确认用户意图
- 追问内容应包含：是新建还是改造现有应用？业务类型是什么？
- 不应在意图不明时直接输出大量无针对性的内容

---

### 测试场景 12：非元服务问题的委托处理

**提示词**：
```
我的元服务里有个崩溃问题，报错是 TypeError: Cannot read property 'xxx' of undefined，怎么调试？
```

**预期输出**：
- AI 代理识别该问题属于通用调试问题，不属于元服务专属问题
- 应委托或引导至合适的调试 Skill 或给出通用 ArkTS 调试建议
- 不应强行将问题归类为元服务专属问题并给出错误指引

---

### 测试场景 13：混合问题处理（合规 + 技术）

**提示词**：
```
我的元服务有两个问题：第一，ArkWeb 在元服务里能用吗？第二，我的包有 5MB，怎么分包？
```

**预期输出**：
- AI 代理识别两个独立问题并分别回答
- 问题 1：明确 `ArkWeb` 不可用，替换为 `AtomicServiceEnhancedWeb`
- 问题 2：说明总包 ≤ 10MB、单分包 ≤ 2MB 的规则，给出分包拆分建议
- 两个问题的回答清晰区分，不混淆

---

### 测试场景 14：支付功能接入咨询

**提示词**：
```
元服务里能不能接入支付功能？用什么 SDK？
```

**预期输出**：
- AI 代理确认元服务支持支付功能
- 指出使用 `@kit.PaymentKit` 进行支付接入
- 提醒需使用带有 `@atomicservice` 标注的支付 API
- 若支付页面涉及 H5，提醒使用 `AtomicServiceEnhancedWeb` 加载

---

### 测试场景 15：与普通 HarmonyOS 应用开发的区别咨询

**提示词**：
```
元服务和普通鸿蒙应用开发有什么区别？我已经会写普通的 HarmonyOS 应用了，切换到元服务开发需要注意什么？
```

**预期输出**：
- AI 代理清晰列出元服务与普通应用的主要区别，至少包含：
  - `bundleType` 配置不同
  - API 范围受限（需 `@atomicservice` 注解）
  - 包大小限制更严格
  - H5 组件不同（`AtomicServiceEnhancedWeb` 替代 `ArkWeb`）
  - 登录方式不同（静默登录）
  - 需要配置可信域名和隐私托管
  - 跳转规范更严格
- 回答针对"已有普通应用开发经验"的背景，突出差异而非从零讲解
