# 测试用例 1：使用 HBuilderX 创建并配置 uni-app 元服务项目

## 场景描述
用户希望从零开始，通过 HBuilderX 开发方式将 uni-app 项目适配为鸿蒙元服务（ASCF），需要了解完整的项目创建和基础配置流程。

## 前置条件
- 已安装 DevEco Studio（最新版本）
- 已安装 HBuilderX 4.34 或更高版本
- 已在华为 AGC（AppGallery Connect）创建元服务应用并获取包名
- 已完成 ICP 备案
- 具备基本的 uni-app 开发经验

## 执行步骤
1. AI 代理解析用户请求："我想用 HBuilderX 把 uni-app 项目适配成鸿蒙元服务，怎么操作？"
2. AI 代理识别用户意图为 HBuilderX 开发方式的完整配置流程
3. AI 代理按以下顺序提供指导：
   - 说明 HBuilderX 版本要求（4.34+）及下载地址
   - 指导在 HBuilderX 中新建或打开 uni-app 项目
   - 指导配置 `manifest.json` 中的鸿蒙包名（与 AGC 中的包名保持一致）
   - 指导配置 `harmony-mp-configs` 节点，包括签名信息和所需权限
   - 说明如何选择"运行到鸿蒙元服务"进行调试

## 预期结果
- AI 代理明确区分 HBuilderX 方式和 CLI 方式，并推荐 HBuilderX 方式适合新手
- 提供 `manifest.json` 中包名配置的具体字段路径和示例值
- 提供 `harmony-mp-configs` 的完整配置示例，包含 `signingConfigs` 和 `permissions` 字段
- 说明签名文件（`.p12`、`.cer`、`profile`）的获取来源（DevEco Studio 自动签名或手动配置）
- 提醒用户包名必须与 AGC 控制台创建的元服务包名完全一致

## 验证方法
- 检查 AI 代理是否提及 HBuilderX 最低版本要求（4.34+）
- 验证配置示例中是否包含 `harmony-mp-configs` 节点结构
- 确认 AI 代理是否说明了签名配置的必要性
- 验证是否提到 AGC 包名一致性的要求

## 备注
HBuilderX 方式是官方推荐的新手入门方式，AI 代理应优先推荐此方式，并在介绍时强调其可视化操作的优势。

# 测试用例 2：使用 CLI 方式配置 Vue3 项目的鸿蒙元服务编译

## 场景描述
用户已有基于 Vue3 的 uni-app CLI 项目，希望通过命令行方式配置鸿蒙元服务的编译构建，并了解版本要求和联合构建脚本配置。

## 前置条件
- 已有 Vue3 uni-app CLI 项目
- Node.js 环境已安装
- 已了解 npm/yarn 基本操作
- 项目中已有 `package.json` 和 `vite.config.js`

## 执行步骤
1. AI 代理解析用户请求："我的 Vue3 uni-app CLI 项目怎么配置鸿蒙元服务编译？需要什么版本？"
2. AI 代理识别用户使用的是 Vue3 CLI 开发方式
3. AI 代理提供以下指导：
   - 说明 Vue3 CLI 所需的 uni-app 版本要求（v3.0.0-4050720250324001 或更高）
   - 提供编译命令：`uni -p mp-harmony`
   - 说明 `package.json` 中的 `scripts` 配置示例
   - 指导配置联合构建脚本（结合 HBuilderX 或 DevEco Studio 构建）
   - 说明产物输出目录（`dist/dev/mp-harmony` 或 `dist/build/mp-harmony`）

## 预期结果
- AI 代理明确给出 Vue3 CLI 的版本要求：`v3.0.0-4050720250324001` 或更高版本（2025年3月24日后的版本）
- 提供 `package.json` 中 scripts 配置示例，包含 `dev:mp-harmony` 和 `build:mp-harmony` 命令
- 说明联合构建的工作原理：uni-app 编译产出 JS/CSS/JSON 资源，DevEco Studio 进行原生编译打包
- 提醒版本不满足时的报错现象和升级方法
- 提供检查当前版本的命令（查看 `package.json` 中 `@dcloudio/uni-app` 版本号）

## 验证方法
- 检查 AI 代理是否给出了明确的版本号要求（含日期标识）
- 验证编译命令 `uni -p mp-harmony` 是否被正确提及
- 确认是否说明了联合构建的两阶段流程
- 验证是否提到 Vue2 和 Vue3 版本要求的区别

## 备注
Vue2 CLI 对应使用 `@dcloudio/uni-mp-harmony`，版本要求为 `v2.0.2-4050720250324001` 或更高。AI 代理应能区分 Vue2 和 Vue3 的不同配置方式。

# 测试用例 3：在 uni-app 项目中使用条件编译适配鸿蒙元服务

## 场景描述
用户希望在已有的多端 uni-app 项目中，仅针对鸿蒙元服务平台编写特定逻辑，保持其他平台代码不受影响，需要了解条件编译的用法。

## 前置条件
- 已有运行中的 uni-app 多端项目（支持微信小程序、H5 等）
- 需要为鸿蒙元服务添加特定的平台逻辑（如使用华为账号登录、访问鸿蒙原生 API）
- 对条件编译概念有基本了解

## 执行步骤
1. AI 代理解析用户请求："我的 uni-app 项目要同时支持微信小程序和鸿蒙元服务，怎么用条件编译区分平台逻辑？"
2. AI 代理识别用户需要条件编译的使用指导
3. AI 代理提供以下内容：
   - 说明鸿蒙元服务的条件编译标识：`MP-HARMONY`
   - 提供 JS/TS 代码中的条件编译示例：`// #ifdef MP-HARMONY` 和 `// #endif`
   - 提供模板（template）中的条件编译示例：`<!-- #ifdef MP-HARMONY -->`
   - 提供样式（style）中的条件编译示例：`/* #ifdef MP-HARMONY */`
   - 说明常见使用场景：华为账号登录按钮、鸿蒙特有权限申请、平台差异化 UI

## 预期结果
- AI 代理给出完整的三种条件编译语法（JS/TS、模板、样式）
- 提供至少一个实际业务场景的代码示例（如华为账号登录按钮仅在鸿蒙平台显示）
- 说明 `MP-HARMONY` 标识的大小写规范（必须全大写）
- 提醒条件编译与运行时判断（`uni.getSystemInfo`）的区别和适用场景
- 给出同时排除某平台的写法：`// #ifndef MP-HARMONY`

## 验证方法
- 检查 AI 代理是否给出了 `#ifdef MP-HARMONY` 的正确写法（大小写）
- 验证是否覆盖了 JS、模板、样式三种语法
- 确认是否提供了实际业务场景的示例代码
- 验证是否说明了 `#ifndef` 的用法

## 备注
条件编译标识在 uni-app 官方文档中为 `MP-HARMONY`，AI 代理不应将其误写为 `HARMONY`、`HarmonyOS` 或其他变体。

# 测试用例 4：排查 uni-app 元服务白屏及网络请求报错问题

## 场景描述
用户在真机调试鸿蒙元服务时遇到白屏问题，同时部分网络请求报错，需要 AI 代理协助排查原因并给出解决方案。

## 前置条件
- 已完成 uni-app 元服务项目的基础配置
- 在 DevEco Studio 中连接真机或模拟器运行
- 遇到白屏（应用启动后页面空白）和接口请求失败的问题

## 执行步骤
1. AI 代理解析用户请求："我的鸿蒙元服务运行后白屏，而且接口请求都报错，怎么排查？"
2. AI 代理识别用户面临两个独立问题：白屏和网络请求报错
3. AI 代理分别给出排查步骤：

   **白屏排查：**
   - 引导使用 `hdc shell hilog` 查看设备日志
   - 说明过滤关键字（如 `[uni-app]`、`ERROR`）
   - 检查是否存在 JS 运行时错误
   - 检查页面路径配置是否正确（`pages.json` 中的首页路径）
   - 检查条件编译是否导致组件缺失

   **网络请求排查：**
   - 说明鸿蒙元服务需要配置域名白名单
   - 指导在 AGC 控制台配置请求域名
   - 说明 HTTP 和 HTTPS 的支持情况

## 预期结果
- AI 代理给出 `hdc shell hilog` 的完整命令，包含过滤参数示例
- 说明白屏的常见原因清单（至少 3 条：JS 错误、路由配置错误、编译产物缺失）
- 明确说明网络请求需要在 AGC 控制台添加域名白名单，而非在代码中配置
- 提供域名白名单的配置路径（AGC 控制台 - 元服务配置 - 服务器域名）
- 提醒开发环境和生产环境的域名需要分别配置

## 验证方法
- 检查 AI 代理是否提到 `hdc shell hilog` 日志查看命令
- 验证是否区分了白屏和网络报错两个问题的排查路径
- 确认是否说明了域名白名单的配置位置（AGC 控制台）
- 验证是否提醒了开发/生产环境域名分别配置

## 备注
`hdc shell hilog` 是鸿蒙设备日志查看的标准命令，AI 代理不应将其与 Android 的 `adb logcat` 混淆。域名白名单是新手最常遗漏的配置项之一。

# 测试用例 5：处理 uni-app 元服务编译报错和签名配置问题

## 场景描述
用户在 DevEco Studio 中构建鸿蒙元服务时遇到多个编译错误，包括签名文件路径错误和 SDK 组件缺失，需要 AI 代理逐一给出解决方案。

## 前置条件
- 已完成 HBuilderX 侧的 uni-app 配置
- 在 DevEco Studio 中进行原生编译时遇到错误
- 错误信息包含 `Invalid storeFile` 或 `Unable to find product 'release'`

## 执行步骤
1. AI 代理解析用户请求："DevEco Studio 编译报错 'Invalid storeFile'，还有 'Unable to find product release'，怎么解决？"
2. AI 代理识别用户遇到签名配置错误和构建变体配置错误
3. AI 代理分别给出解决方案：

   **Invalid storeFile 错误：**
   - 检查 `harmony-mp-configs` 中 `signingConfigs.storeFile` 路径是否为绝对路径
   - 检查 `.p12` 文件是否存在于指定路径
   - 说明签名文件的正确获取方式（DevEco Studio 自动管理签名）

   **Unable to find product 'release' 错误：**
   - 检查 DevEco Studio 项目中是否配置了 `release` 构建变体
   - 说明在 `build-profile.json5` 中添加 release 配置的方法
   - 提醒 HBuilderX 发布时默认使用 release 构建，需确保签名配置完整

4. AI 代理额外说明 SDK 组件缺失的处理方式

## 预期结果
- AI 代理对每个错误信息给出针对性的根因分析
- 提供 `signingConfigs` 配置的正确示例（包含 `storeFile`、`storePassword`、`keyAlias`、`keyPassword` 字段）
- 说明 `build-profile.json5` 中 release 构建变体的配置结构
- 说明 SDK 组件缺失时通过 DevEco Studio SDK Manager 安装缺失组件的步骤
- 提醒密码字段不应包含特殊字符或空格

## 验证方法
- 检查 AI 代理是否分别针对每个错误给出了独立的解决方案
- 验证 `signingConfigs` 配置示例是否字段完整
- 确认是否提到 `build-profile.json5` 文件的修改方式
- 验证是否说明了 SDK Manager 的使用场景

## 备注
`Invalid storeFile` 是最常见的签名配置错误，通常由路径使用相对路径或反斜杠（Windows 路径）导致。AI 代理应提醒使用绝对路径并注意路径分隔符。

# 测试用例 6：处理华为账号登录集成问题及上架审核驳回

## 场景描述
用户完成了元服务开发并提交上架，审核被驳回，原因包括：华为账号登录失败、隐私政策缺失、图标不规范。同时用户在集成华为账号登录时遇到签名和 client_id 配置问题。

## 前置条件
- 已完成元服务基础开发
- 已提交 AGC 审核但被驳回
- 驳回原因包含多个问题项
- 需要修复后重新提交审核

## 执行步骤
1. AI 代理解析用户请求："我的元服务审核被驳回了，原因是华为账号登录报错、没有隐私政策、图标不规范，怎么逐一修复？"
2. AI 代理识别用户面临三个独立的审核驳回问题
3. AI 代理按优先级给出修复指导：

   **华为账号登录修复：**
   - 检查 `client_id` 是否与 AGC 控制台中元服务的应用 ID 一致
   - 检查调试签名和发布签名的 SHA256 是否已在 AGC 控制台添加
   - 说明正式包必须使用发布签名，不能使用调试签名提交审核

   **隐私政策配置：**
   - 说明需要在 `manifest.json` 或元服务配置中添加隐私政策 URL
   - 提醒隐私政策页面必须可正常访问（HTTPS）
   - 说明隐私政策内容需符合鸿蒙/华为审核要求

   **图标规范：**
   - 说明鸿蒙元服务图标尺寸要求（216x216 px，圆角遮罩）
   - 提醒图标不能包含文字、不能与应用市场图标高度相似
   - 提供图标设计规范的参考链接

## 预期结果
- AI 代理对三个审核驳回问题分别给出独立的修复方案
- 华为账号登录问题明确指出 `client_id` 和签名 SHA256 是最常见的易错点
- 说明 AGC 控制台中添加 SHA256 的具体路径
- 隐私政策说明需要包含数据收集、存储、使用等核心条款
- 图标规范提供具体尺寸数值和设计限制说明

## 验证方法
- 检查 AI 代理是否提到 `client_id` 与 AGC 应用 ID 一致的要求
- 验证是否区分了调试签名和发布签名的使用场景
- 确认是否给出了图标尺寸的具体数值
- 验证是否说明了隐私政策的可访问性要求（HTTPS）

## 备注
华为账号登录的签名和 `client_id` 配置是上架阶段最高频的问题，AI 代理应重点强调发布签名与调试签名的区别，避免用户将开发阶段的调试包配置直接用于发布。
