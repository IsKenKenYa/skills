## 4. 装饰器用在错误的位置

AI 经常混淆装饰器应该装饰什么。

| ❌ AI 常见错误 | 原因 | 规则章节 |
|--------------|------|---------|
| `@State` 装饰组件入参 | @State 只能装饰组件内部变量，入参用 @Prop/@Param | 规则第3节 状态管理V1装饰器, 规则第4节 状态管理V2装饰器 |
| `@Local` 装饰组件入参 | @Local 只能组件内部初始化，入参用 @Param | 规则第4节 状态管理V2装饰器 |
| `@Prop` 装饰组件内部变量 | @Prop 只能装饰组件入参 | 规则第3节 状态管理V1装饰器 |
| `@Param` 装饰组件内部变量 | @Param 只能装饰组件入参 | 规则第4节 状态管理V2装饰器 |
| `@Link` 本地初始化 `@Link count: number = 0` | @Link 禁止本地初始化，必须外部传入 | 规则第3节 状态管理V1装饰器 |
| `@ObjectLink` 本地初始化 | @ObjectLink 禁止本地初始化 | 规则第3节 状态管理V1装饰器 |
| `@ObjectLink` 装饰简单类型 | @ObjectLink 不支持 number/string/boolean | 规则第3节 状态管理V1装饰器 |
| `@Consume` 被外部初始化 | @Consume 不可以被外部初始化 | 规则第3节 状态管理V1装饰器 |

**根因**：AI 把装饰器理解为通用的"响应式标记"，不理解初始化方向和位置约束。
