# ArkUI 代码检视清单

> 每条标注了对应的 `quick-rules/` 章节文件号，方便速查。各文件末尾的「常见错误」章节包含详细错误模式。

## import 和模块

- [ ] 所有 import 使用 `@kit.*` 路径，不存在 `@ohos.*`（见规则第20节 模块导入，错误清单第1节）
- [ ] import 的符号名和所属 kit 正确（见规则第20节 Kit导入速查表，错误清单第1节）
- [ ] 不存在 import 了但未使用的模块（见规则第20节 模块导入）
- [ ] 不存在遗漏的 import（使用的外部类型/函数/枚举均有对应声明）（见规则第20节 模块导入）

## UIContext 和全局接口

- [ ] router、promptAction、AlertDialog、animateTo、vp2px 等全局接口均通过 `this.getUIContext()` 调用（见规则第19节 UIContext替换，错误清单第2节）

## 状态管理

- [ ] 所有状态变量是否声明了类型（见规则第3/4节 状态管理V1/V2装饰器，错误清单第10节）
- [ ] 每个装饰器用在正确的位置：@State/@Local 装饰组件内变量，@Prop/@Param 装饰组件入参（见规则第3/4节 状态管理V1/V2装饰器，错误清单第4节）
- [ ] 避免状态管理 V1 与 V2 混用（见规则第5节 V1/V2混用，错误清单第3节）
- [ ] @Link / @ObjectLink 不存在本地初始化（见规则第3节 状态管理V1装饰器，错误清单第4节）

## 渲染控制

- [ ] ForEach / LazyForEach 是否有第三个参数作为 key，且用业务 ID 不用索引（见规则第7节 渲染控制，错误清单第6节）

## build() 函数

- [ ] build() 中不存在本地变量声明、console.info、switch、三元表达式（见规则第2节 build函数，错误清单第5节）
- [ ] build() 中不直接改变状态变量（见规则第2节 build函数，错误清单第5节）

## 导航

- [ ] 路由跳转使用 Navigation 实现，不使用 Router（见规则第9节 导航与路由，错误清单第8节）
- [ ] 多页面Navigation需要考虑子页面是否有注册页面

## 命名和嵌套

- [ ] 变量名避开框架保留字（见规则第21节 内置保留字，错误清单第9节）
- [ ] 组件和变量命名避开框架内置组件和属性名（见规则第1节 自定义组件基本约束，错误清单第9节）
- [ ] 组件嵌套符合约束（如 List 内必须用 ListItem，Tabs 内必须用 TabContent）（见规则第14节 布局容器，错误清单第9节）

## 证据和废弃

- [ ] API 参数与 quick-apis 表或检索结果一致，未凭记忆编造（见错误清单第7节）
- [ ] 关键 API 均有 quick-apis 表确认或检索证据路径支撑
- [ ] 不使用废弃接口（CustomDialogController、pageTransition 等）（见规则第8/10节 动画/对话框，错误清单第11节）
- [ ] 证据不足的 API 已标注"待确认"，未强行使用
- [ ] 多方案冲突时已同时给出推荐与备选

## 代码风格（参考 `style-guide.md`）

- [ ] 组件/变量命名遵循 style_guide 命名规范（PascalCase 组件、camelCase 变量）
- [ ] 组件内部结构顺序：状态定义 → 属性 → 私有成员 → 生命周期 → 私有方法 → build()
- [ ] 链式调用每个方法一行，逻辑语句加分号，UI 语句不加分号
