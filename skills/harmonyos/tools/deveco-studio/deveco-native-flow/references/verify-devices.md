# UI 验证设备矩阵详细说明

> 本文档是从 SKILL.md 移出的详细设备矩阵说明，用于多端多设备验证。

---

## HarmonyOS 设备矩阵

| 顺序 | 设备类型 | 验证重点 | 优先级 |
|------|---------|---------|--------|
| 1 | 手机 | 基础布局、核心交互 | 必选 |
| 2 | 阔折叠 | 分栏/多窗适配 | 推荐 |
| 3 | 折叠屏 | 折叠切换适配 | 推荐 |
| 4 | 三折叠 | 多折适配 | 可选 |
| 5 | 平板 | 平板布局适配 | 推荐 |

**最低要求**：至少在手机设备上验证。推荐手机 + 至少一种大屏设备。

## 验证流程详解

### 1. 选择验证设备

按设备矩阵选择至少一种设备类型（手机必选）：

```
最低要求：手机设备验证（必选）
推荐配置：手机 + 至少一种大屏设备（折叠/平板）
可选配置：全部设备类型覆盖
```

### 2. 安装构建产物

将构建产物安装到目标设备：

| 平台 | 安装方式 |
|------|---------|
| HarmonyOS | `hdc install {hap_path}` 或 MCP start_app |
| Android | `adb install {apk_path}` |
| iOS | Xcode 安装或 simctl |

### 3. 逐 UserStory 验证

按 plan 中的 UserStory，逐设备类型执行：

1. **启动应用**：安装后启动进入主界面
2. **执行操作步骤**：按 UserStory 步骤逐个执行
3. **截图验证**：关键步骤截图保存
4. **日志验证**：获取应用日志确认功能执行
5. **UI 结构验证**：获取 UI 树确认元素存在

### 4. 汇总验证报告

生成 `verification-report-{platform}.md`：

```markdown
# 验证报告：{platform}

## 设备信息
- 设备类型：{device_type}
- 设备名称：{device_name}
- 验证时间：{timestamp}

## UserStory 验证结果

### US-1: {story_name}
- 状态：✅ 通过 / ❌ 失败
- 截图：{screenshot_paths}
- 日志：关键日志片段
- 备注：{notes}

### US-2: ...

## 总体结论
- 通过率：XX%
- 建议修复项：{issues}
```

---

## 失败处理

- **全部通过** → 更新 state.json：completed_phases 添加 "verify"，流程结束
- **存在失败** → 输出验证报告，询问用户：
  - **fix**：回到编码修复（重新执行 coding）
  - **ignore**：标记已知问题，记录到 metadata.json
  - **retry**：重新执行失败项

## 特殊设备验证要点

### 折叠屏设备

- 验证折叠状态切换时 UI 自适应
- 验证分栏布局在不同折叠状态下的表现
- 检查折叠事件监听是否正确触发

### 三折叠设备

- 验证三种折叠模式的 UI 呃应
- 检查多屏协同功能（如有）
- 确认三屏模式下的布局合理性

### 平板设备

- 验证大屏布局是否合理利用空间
- 检查横竖屏切换适配
- 确认分屏模式下的显示效果
